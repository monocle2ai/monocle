# Monocle Linter - Validate Trace Conformance

The Monocle Linter is a lightweight validation system that checks whether Monocle traces follow good observability conventions laid out in the metamodel (similar to how Spectral validates OpenAPI specs).

## Overview

When your GenAI application instruments code with Monocle, it generates trace files (JSON) containing spans. The linter validates these traces against the Monocle metamodel to ensure they conform to observability best practices.

## Features

- ✅ **Required Fields Validation** - Ensures spans have mandatory attributes
- ✅ **Token Count Validation** - Detects missing token counts in inference spans
- ✅ **Tool Metadata Validation** - Validates tool invocation metadata
- ✅ **Naming Convention Validation** - Enforces snake_case naming for spans
- ✅ **Memory Caching** - Specs are loaded once and cached in memory
- ✅ **Configurable Strictness** - basic, selective, or strict validation levels

## Installation

The linter is included with monocle-apptrace:

```bash
pip install monocle_apptrace
```

## CLI Usage

### Basic Command

```bash
monocle-apptrace validate trace.json
```

### With Options

```bash
# Strict validation (all span types)
monocle-apptrace validate trace.json --level strict

# Selective validation (inference + tool spans only, default)
monocle-apptrace validate trace.json --level selective

# Basic validation (just span.type present)
monocle-apptrace validate trace.json --level basic

# Treat warnings as errors
monocle-apptrace validate trace.json --fail-on-warning

# Combine options
monocle-apptrace validate trace.json --level strict --fail-on-warning
```

### Exit Codes

- **0** : All spans are valid
- **1** : Validation errors found OR warnings found (with `--fail-on-warning`)

### Output Examples

**Valid trace:**
```
✅ All spans are valid!
```

**Invalid trace:**
```
ERROR: missing required field entity.2.name
ERROR: missing prompt_tokens in metadata
WARNING: span name should be snake_case, got 'OpenAI_ChatCompletion_Create'
```

## Validation Rules

### 1. Required Fields Rule

**What it validates:** Ensures spans have mandatory attributes

**Applies to:** All inference spans

**Required fields:**
- `entity.2.name` - Model name (e.g., "gpt-4")
- `entity.2.type` - Model type (e.g., "model.llm")

**Example error:**
```
ERROR: missing required field entity.2.name
```

### 2. Token Count Rule

**What it validates:** Detects missing token counts in metadata

**Applies to:** Inference spans

**Logic:**
- If provider returns tokens (OpenAI, Anthropic, Gemini, etc.) → **ERROR** if missing
- If provider doesn't support tokens → **WARNING** if missing

**Required fields (for supported providers):**
- `prompt_tokens` - Tokens sent to model
- `completion_tokens` - Tokens generated by model

**Example error:**
```
ERROR: missing prompt_tokens in metadata
```

**Example warning:**
```
WARNING: missing prompt_tokens in metadata (optional for this provider)
```

### 3. Tool Metadata Rule

**What it validates:** Ensures tool invocations have required metadata

**Applies to:** Agentic tool invocation spans (`span.type=agentic.tool.invocation`)

**Required fields:**
- `tool.status` - Execution status (success, error, timeout)
- `tool.return_value` - Value returned by the tool

**Example error:**
```
ERROR: missing tool.status in tool metadata
```

### 4. Naming Convention Rule

**What it validates:** Enforces snake_case naming convention

**Applies to:** All spans

**Valid patterns:** `snake_case`, with dots and underscores allowed
- ✅ `openai.create`
- ✅ `langchain_rag_retrieve`
- ✅ `search_tool`

**Invalid patterns:**
- ❌ `OpenAI.Create` (uppercase)
- ❌ `langchain-rag` (hyphens)
- ❌ `SearchTool` (camelCase)

**Example warning:**
```
WARNING: span name should be snake_case, got 'OpenAI_ChatCompletion_Create'
```

## Programmatic Usage

### Basic Validation

```python
from monocle_apptrace.linter import MonocleValidator
from pathlib import Path

# Initialize validator
validator = MonocleValidator()

# Validate trace file
results = validator.validate_trace_file(Path("trace.json"))

# Check results
if results.has_errors():
    print(f"Found {len(results.all_errors)} errors")
    for error in results.all_errors:
        print(f"  - {error.message}")

if results.has_warnings():
    print(f"Found {len(results.all_warnings)} warnings")
    for warning in results.all_warnings:
        print(f"  - {warning.message}")
```

### Validate Single Span

```python
from monocle_apptrace.linter import MonocleValidator

validator = MonocleValidator()

span = {
    "name": "openai.create",
    "attributes": {
        "span.type": "inference",
        "entity.2.name": "gpt-4",
        "entity.2.type": "model.llm"
    },
    "events": [{
        "name": "metadata",
        "attributes": {
            "prompt_tokens": 10,
            "completion_tokens": 5
        }
    }]
}

errors = validator.validate_span(span)
print(f"Found {len(errors)} validation issues")
```

### Custom Validation with Reporter

```python
from monocle_apptrace.linter import MonocleValidator, ValidationReporter
from pathlib import Path

validator = MonocleValidator()
results = validator.validate_trace_file(Path("trace.json"))

reporter = ValidationReporter()
output = reporter.format_results(results, fail_on_warning=False)
print(output)

exit_code = reporter.get_exit_code(results, fail_on_warning=True)
sys.exit(exit_code)
```

## Trace JSON Structure

The linter expects traces in the following JSON format:

```json
[
  {
    "name": "openai.ChatCompletion.create",
    "context": {
      "trace_id": "0x7d675c7ccfcbecbc581510d4873bedca",
      "span_id": "0x543b6c7c35b4f0bf"
    },
    "attributes": {
      "span.type": "inference",
      "entity.1.provider_name": "openai",
      "entity.1.type": "inference.openai",
      "entity.2.name": "gpt-4",
      "entity.2.type": "model.llm",
      "workflow.name": "my_app"
    },
    "events": [
      {
        "name": "data.input",
        "attributes": {
          "input": ["Tell me a joke"]
        }
      },
      {
        "name": "metadata",
        "attributes": {
          "prompt_tokens": 12,
          "completion_tokens": 18,
          "finish_reason": "stop"
        }
      }
    ]
  }
]
```

## Supported Providers (for Token Count Validation)

Providers that return token counts (requires tokens in trace):
- OpenAI
- Anthropic
- Google Gemini
- AWS Bedrock
- Azure OpenAI

Other providers: token counts are optional (warnings only)

## Specs Loading

The linter automatically loads validation specifications from the [monocle-specs](https://github.com/monocle2ai/monocle-specs) repository:

```
https://raw.githubusercontent.com/monocle2ai/monocle-specs/main/metamodel/entities/
```

Specs are cached in memory after first load for performance.

### Available Spec Files

- `entities.json` - Entity type definitions
- `inference_types.json` - Inference service types
- `model_types.json` - Model type specifications
- `vector_store_types.json` - Vector store types
- `app_hosting_types.json` - Application hosting types
- `workflow_types.json` - Workflow component types

## Architecture

### Modules

- **`specs_loader.py`** - Loads and caches specs from monocle-specs
- **`rules.py`** - Defines validation rules (RequiredFieldRule, TokenCountRule, etc.)
- **`validator.py`** - Main validator that applies rules to spans
- **`reporters.py`** - Formats validation results as ERROR/WARNING messages

### Class Hierarchy

```
Rule (abstract base class)
├── RequiredFieldRule
├── TokenCountRule
├── ToolMetadataRule
└── NamingConventionRule

MonocleValidator
├── specs (loaded via SpecsLoader)
└── rules (built from specs)

ValidationResult
├── errors
├── all_errors
└── all_warnings

ValidationReporter
├── format_results()
└── get_exit_code()
```

## Performance

- **First call**: ~500ms (loads specs from GitHub)
- **Subsequent calls**: ~1ms (uses cached specs in memory)

The specs are cached in memory using a class variable, so:
- Single validator instance: specs loaded once
- Multiple validators: each loads specs once independently

To clear cache (useful for testing):

```python
from monocle_apptrace.linter import SpecsLoader
SpecsLoader.clear_cache()
```

## Common Issues

### Issue: "missing required field entity.2.name"

**Cause:** Your inference span doesn't have the model name

**Solution:** Ensure the span's metadata includes `entity.2.name`:

```json
{
  "attributes": {
    "span.type": "inference",
    "entity.2.name": "gpt-4",  // Add this
    "entity.2.type": "model.llm"
  }
}
```

### Issue: "missing prompt_tokens in metadata"

**Cause:** Token counts are missing for a provider that returns them

**Solution:** Ensure metadata event includes token counts:

```json
{
  "events": [{
    "name": "metadata",
    "attributes": {
      "prompt_tokens": 10,      // Add this
      "completion_tokens": 5    // Add this
    }
  }]
}
```

### Issue: "span name should be snake_case"

**Cause:** Span name contains uppercase letters or invalid characters

**Solution:** Use lowercase with underscores/dots:

```python
# ❌ Wrong
"name": "OpenAI.ChatCompletion.Create"

# ✅ Correct
"name": "openai.chatcompletion.create"
```
